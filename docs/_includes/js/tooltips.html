可选择的加入功能

鉴于性能方面，Tooltip（工具提示）以及Popover（弹框提示）组件的data属性api是可选择的，即你必须手动初始化他们

可以通过data-toggle属性选择一个页面中的所有tooltips，并对其进行初始化：

{% highlight js %} $(function () { $('[data-toggle="tooltip"]').tooltip() }) {% endhighlight %}

Tooltips与按钮组和输入框组联合使用时需要一些特殊的设置

在.btn-group或.input-group使用Tooltips时,需要指定container: 'body'选项()，目的是避免不必要的副作用(比如，当Tooltip显示之后，与其合作的页面元素可能变得更宽或者失去圆角）。

不要在隐藏元素上面使用tooltips

当目标元素被设置为display: none;时会引发$(...).tooltip('show')的定位异常。

在页面禁止的元素上面使用Tooltips时，需要用一个元素将其包裹起来

为一个disabled or .disabled元素添加Tooltips时, 需要将这个元素放在<div>之内，并将tooltip附加在<div>元素上。

使用

Tooltip一经请求将自动生成内容和标记，并且在元素触发之后会自动定位其显示位置。

通过Javascript代码触发Tooltip：

{% highlight js %} $('#example').tooltip(options) {% endhighlight %}

标记

仅仅需要在你想要其拥有Tooltip的元素上面添加data属性和title属性即可。生成的tooltip是相当简单的，尽管其确实需要一个位置(默认情况下，该位置被设置为top)。

换行的链接

Sometimes you want to add a tooltip to a hyperlink that wraps multiple lines. The default behavior of the tooltip plugin is to center it horizontally and vertically. Add white-space: nowrap; to your anchors to avoid this.

选项

可以通过data属性或者javascript设置选项，对于data属性，可以通过在选项前面添加data-前缀的方式进行配置，比如data-animation=""。

名称 类型 默认 Description
animation boolean true 为tooltip添加fade的过渡效果
container string | false false

为指定元素添加Tooltip。比如container: 'body'。该选项允许你在触发元素附近的文档流中定位Tooltip， 这在窗口调整大小时防止Tooltip浮动而远离触发元素，这种情况下非常有用。
delay number | object 0

延迟tooltip的展示和隐藏(ms为单位)，记住该选项在手动触发的tooltip上面无效果。

如果提供了一个数字，则tooltip的展示和隐藏都将延迟指定的时间

对象类型为：delay: { "show": 500, "hide": 100 }
html boolean false 向tooltip中插入一段HTML。如果设置为false，jQuery的text方法将被用来向DOM中插入内容。如果你担心XSS攻击的话，使用文本。
placement string | function 'top'

该选项用来设置tooltip的展示位置 - top | bottom | left | right | auto。
当设定为auto时，组件将动态定位tooltip的展示位置。比如，如果该选项设定为“auto left”，tooltip将显示在元素的左侧，如果左侧空间不足则显示在右侧。

当该选项被设定为一个函数时，函数的第一个参数为tooltip DOM元素，第二个参数为触发tooltip的DOM元素。上下文的this被设定为tooltip实例。
selector string false 如果提供了一个选择器，Tooltip对象将被委托给指定的目标元素。实际上，这主要作用是使动态插入的HTML内容中可以加入Tooltip。可以查看这里以及一个例子.
template string '<div class="tooltip" role="tooltip"><div class="tooltip-arrow"></div><div class="tooltip-inner"></div></div>'

创建tooltip时使用的基本HTML。

tooltip的title属性将被填充到.tooltip-inner中。

.tooltip-arrow将展示tooltip的箭头。

最外层包裹的元素应该具有.tooltip类
title string | function ''

在title属性没有设定下的默认展示title值。

如果该选项被设定为一个函数，那么该函数作用域中的this指向tooltip元素。
trigger string 'hover focus' 该选项用来设定tooltip的触发方式 - click | hover | focus | manual。组件允许传入多个触发事件，使用space进行间隔即可。
viewport string | object { selector: 'body', padding: 0 }

保持tooltip在元素的边界内。比如viewport: '#viewport'或者{ "selector": "#viewport", "padding": 0 }

使用Data属性定义单个的tooltip

单个的tooltip可以通过data属性指定以上选项。

方法

$().tooltip(options)

将tooltip绑定到一个元素集合上。

.tooltip('show')

显示一个元素的tooltip。titlt宽度为0的tooltip永远不会被展示。

{% highlight js %}$('#element').tooltip('show'){% endhighlight %}

.tooltip('hide')

隐藏一个元素的tooltip。

{% highlight js %}$('#element').tooltip('hide'){% endhighlight %}

.tooltip('toggle')

切换tooltip的展示和隐藏。

{% highlight js %}$('#element').tooltip('toggle'){% endhighlight %}

.tooltip('destroy')

隐藏并销毁一个元素的tooltip.

{% highlight js %}$('#element').tooltip('destroy'){% endhighlight %}

Events

事件类型 描述
show 当且仅当show实例方法被调用时该方法被触发。
shown 当且仅当该tooltip可见（等待CSS过渡效果完成之后）时触发该事件。
hide 当且仅当hide实例方法被调用时该方法被触发。
hidden 当且仅当该tooltip完全隐藏（等待CSS过渡效果完成之后）后触发该事件。
{% highlight js %} $('#myTooltip').on('hidden', function () { // do something… }) {% endhighlight %}
